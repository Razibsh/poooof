# Backlog
<!-- poooof X.Y.Z -->

> Every idea lands here the moment it comes up — captured by the agent automatically, without
> being asked. Kept organized on its own: **dated, newest first, grouped by status**, area-tagged.
> Nothing is ever deleted — items move down the lifecycle (Inbox → Promoted → Done). Nothing gets
> built out of roadmap order; nothing gets forgotten.
>
> Format: `- YYYY-MM-DD [area] idea — context`  ·  newest at the top of each section.
> Run `/poooof:tidy` any time to re-sort, date, and de-duplicate.

## 📥 Inbox — newest first

- YYYY-MM-DD [example] <idea> — <why it came up / any context>

## 🔨 Promoted to a phase

> When a backlog item becomes a real phase, move it here with a dated arrow noting which phase —
> so the backlog stays a record of decisions, not just a wishlist.

- YYYY-MM-DD [example] <idea> — <context> → Roadmap Phase N (YYYY-MM-DD)

## ✅ Done / shipped

> Delivered items land here with a dated arrow (which release/phase shipped them). History, kept.

- YYYY-MM-DD [example] <idea> — <context> → shipped <version/phase> (YYYY-MM-DD)
